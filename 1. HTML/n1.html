1. HTML es el lenguaje de marcado principal de la WWW. Originalmente fue diseñado como un lenguaje para describir semánticamente documentos científicos, aunque con el paso del siempo se ha adaptado para describir otro tipo de documentos e incluso aplicaciones.

Fue creado en el añi 1990, aunque su primera publicación formal fue en 1991 y su primer lanzamiento

fue en 1992. Desde entonces han salido diferentes versiones:

  • HTML 1.0, lanzado en 1993. Contenía un conjunto básico de etiquetas.
  • HTML 2.0, en 1995. Es el primer estándar oficial de HTML. Añadió soporte para formularios y scripts.
  • HTML 3.2, en 1997. Agregó soporte para tablas, marcos y hojas de estilo en línea.
  • HTML 4.0, en 1997. Agregó soporte para hojas de estilo externas, capas y marcos en línea.
  • HTML 4.1, en 1999. Corrigió errores de la versión anterior.
  • XHTML 1.0, en 2000. Se basó en la sintaxis de XML y proporcionó mayor modularidad y estructura al lenguaje.
  • HTML 5, en 2014. Introduce características nuevas, como soporte multimedia. Es la última versión de HTML.

2. Los principios básicos que el W3C recomienda seguir para la creación de documentos HTML son:

  • Separar estructura y presentación para facilitar las revisiones del documento y recurdir el coste de servir a un amplio espectro de plataformas.
  • Considerar la accesibilidad universar a la Web, en especial para aquellos con discapacidades.
  • Ayudar a los agentes de usuario con la representación incremental mediante un diseño cuidadoso de tablas y haciendo uso de las características más actualizadas.

3. Un elemento o atributo se considera desaprobado cuando ha quedado anticuado por la presencia de estructuras nuevas. Se definen en el manual de referencia, pero son marcados como desaprobados. Estos elementos pueden ser posteriormente declarados como obsoletos en futuras versiones de HTML.

Un elemento o atributo obsoleto es quel para el cual no hay garantía de soporte por parte de un agente usuario. No se encuentran definidos en la especificación, pero sí enumerados por motivos históricos.

4. El DTD (Document Type Declaration - Definición de Tipo de Documento) es la sección que se ubica en la primera línea del archivo HTML. Esta permite saber al navegador qué especificación estamos utilizando.

En la especificación HTML 4.01, podemos declararla como:

  • Declaración transitoria: !DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"
  • Declaración estricta: !DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"

5. Los metadatos son datos que describen el contenido informativo de un documento. En HTML, la especificación de metadatos implica dos pasos:

  • Declaración de una propiedad y de un valor para esta propiedad. Esto se puede hacer dentro del documento por medio del elemento META, o desde fuera vinculando los metadatos por medio del elemento LINK.
  • Referencia a un perfil en el que se definen la propiedad y sus valores legales. Para designar un perfil, se usa el tributo "profile" del elemento HEAD.
